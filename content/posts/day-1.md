Focused on the fundamentals of Git and GitHub and how code is managed over time.

I learned how to track changes, create commits, and push code to a repository so it’s stored, versioned, and easy to share. Instead of working with scattered files, everything lives in one place with a clear history.

What clicked for me was the workflow:

write code → commit changes → push to GitHub

Along with that, I started understanding best practices that make this process more effective:
- Keep commits small and focused
- Write clear commit messages
- Organize files so the project is easy to navigate
- Use .gitignore to avoid uploading unnecessary or sensitive files

We also talked about AI-specific habits, especially being careful with what you paste into tools and making sure not to expose things like API keys or sensitive data.

This week is about getting back to the fundamentals with more intention.

Not just using tools, but understanding how to use them responsibly and effectively.
