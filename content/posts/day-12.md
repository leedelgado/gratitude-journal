title: "Day 12: The Scientific Method of Quality Assurance"
date: "2026-04-25"
---

Today was my first real deep dive into quality assurance, and it changed how I think about building products. Before this, I thought fixing bugs was just about spotting what’s broken and correcting it. Now I understand it’s a full process that requires structure and intention.

I learned the difference between QA, testing, debugging, and troubleshooting. QA is about preventing problems from happening in the first place. Testing is about trying to break the app and finding issues. Debugging is figuring out the root cause of those issues. Troubleshooting is fixing the problem and making sure everything works again.

What stood out the most is how small mistakes in code can create big problems in the user experience. Something as simple as hardcoding a value or using the wrong operator can completely break how the app behaves. It made me realize how important it is to slow down and think through what the code is actually doing.

When I worked through the bug, I followed a real process. I observed the issue, made a guess about what was causing it, tested that idea, and then fixed it. That showed me that debugging is not random. It’s a step-by-step way of thinking, almost like using the scientific method.

I also learned how important it is to test beyond the normal flow. Real users don’t always do things the “right” way, so I need to test different scenarios, wrong inputs, and edge cases. That mindset helps uncover issues that wouldn’t show up otherwise.

Another key takeaway was regression testing. Fixing a bug is not enough. I need to make sure that my fix didn’t break anything else. That means going back and testing related features to confirm everything still works as expected.

Overall, today helped me shift from just trying to build things to actually thinking about how to build reliable, high-quality products.

I’m grateful that I’m learning how to think through problems instead of guessing, and that I’m building real skills that will help me create better and more reliable experiences.
